Method Detail
authorize()method
public static function authorize(callback:Function, view:StageWebView, apiKey:String, secretKey:String, redirect_uri:String, scopes:Array):void

We use OAuth 2.0 for authorize the user into the app, please review the LinkedIn Developer API website for more information.

Parameters

callback:Function — (Required) Handle method for the response of the server
 
view:StageWebView — (Required) StageWebView object that will show the LinkedIn Authorization and Login page
 
apiKey:String — (Required) Value of your API Key given when you registered your application with LinkedIn
 
secretKey:String — (Required) Value of your secret key given when you registered your application with LinkedIn
 
redirect_uri:String — (Required) Same redirect_uri that you passed in the previous step.
 
scopes:Array — (Required) Array with the scopes. Use it to specify a list of member permissions that you need and these will be shown to the user on LinkedIn's authorization form. If not passed, defaults to r_ basicprofile. Specify multiple scopes by adding a space between different parameters. Remember to URL encode your spaces! Example: ["r_fullprofile", "r_emailaddress", "r_network"].
