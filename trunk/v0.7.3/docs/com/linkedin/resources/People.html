InheritancePeople Object

Tap into LinkedIn's 200M+ member base to power professional identity and networking features within your application. LinkedIn's profile APIs allow you to:
  • Leverage LinkedIn as an identity authority for application registration and signin with access to name, photo, headline, and email address
  • Light up the identity of your user inside your application, with attribution to LinkedIn
  • Enable your user to leverage their 1st degree connections for personal use within your application
  • Find and connect with other LinkedIn members in your user's 2nd degree network using our Search API

    byId()method
    public static function byId(callback:Function, userId:String, fields:Array = null):void

    Returns a member's LinkedIn profile

    Parameters

    callback:Function — (Required) Handle method for the response
     
    userId:String — (Required) Use the tilde character to indicate that you want the profile of the current user. If you know the member token of the profile you want to display, you can use it as the key value for this API call.
     
    fields:Array (default = null) — (Optional) Profile Fields. Remember, this can change depend of the scope that will use to load the user information. Review the fields here: link https://developer.linkedin.com/documents/profile-fields


    Example
    Request information for the current user People.byId(userReady, "~");
    Example
    Request information for a user People.byId(userReady, "abcdefg");
    Example
    Request information using fields People.byId(userReady, "abcdefg", ["first-name", "headline"]);
    connections()method 
    public static function connections(callback:Function, userId:String, fields:Array = null, params:Object = null):void

    The Connections API returns a list of 1st degree connections for a user who has granted access to his/her account. Remember you need the "r_network" permission.

    Parameters

    callback:Function — (Required) Handle method for the response
     
    userId:String — (Required) User ~ if you want the connections for the current user or put the ID like "abcdef"
     
    fields:Array (default = null) — (Optional) Connections Fields. Since the objects returned are of the "person" type you can access any of the Profile Fields available with the r_basicprofile member permission.
     
    params:Object (default = null) — (Optional) If you don't specify any parameters in the URL, you get full details of all connections. Misspelled parameter names are ignored. You can see what params you can use in the link.


    Example
    If you use the current user: People.connections(conReady, "~")
    Example
    If you use the current user, with fields and params: People.connections(conReady, "~", ["first-name"], {"count": "10", "modified": "new"})

    search()method 
    public static function search(callback:Function, q:Object):void

    The People Search API returns information about people. It lets you implement most of what shows up when you do a search for "People" in the top right box on LinkedIn.com. WARNING! People Search API is a part of our Vetted API Access Program. You must apply https://help.linkedin.com/app/ask/path/api-dvr and get LinkedIn's approval before using this API.

    Parameters

    callback:Function — (Required) Handle method for the response
     
    q:Object — (Required) Object with the parameters for the search. Check the link for know whats parameters you can use.


    Example
    People.search(searchReady, {"keywords": "Developer Android", "first-name": "Patricio"})
